# 질문 형식 예시 (Step 3)

**질문 포맷:** 구조화된 질문 포맷을 사용한다 — 번호 매긴 선택지 + ★ 권장 옵션 표기.

각 질문에 선택지 4~5개 + 권장 옵션(★)을 반드시 표시한다.

## 기본 형식 (필수 3항목)

```markdown
## 진행 전에 확인할 부분이 있어요

1. 이 문서로 무엇을 해드리면 될까요?
   - (a) 요약
   - (b) 분석 ★
   - (c) 검토
   - (d) 보고서 작성
   - (e) 기타 (직접 입력)

2. 어디에 집중하면 될까요?
   - (a) 전체
   - (b) 특정 챕터/섹션
   - (c) 특정 관점 (예: 비용, 리스크, 일정)
   - (d) 기타 (직접 입력)

3. 결과는 어떤 형태가 좋을까요?
   - (a) 짧은 요약 (1페이지 이내)
   - (b) 표/비교표
   - (c) 상세 보고서 ★
   - (d) 슬라이드
   - (e) 기타 (직접 입력)
```

## 배경 질문 추가 (#4 필수 케이스)

요청 유형이 **분석·전략 제안·보고서 작성·평가·리뷰**일 때 아래를 추가한다:

```markdown
4. 이 결과물은 누구를 위한 것인가요?
   - (a) 내부 팀 공유
   - (b) 임원 보고 ★
   - (c) 외부 고객 전달
   - (d) 개인 참고용

5. (필요 시) 참고할 시점·기준·비교 대상이 있나요?
   - (a) 최근 분기 데이터만
   - (b) 전년 대비
   - (c) 경쟁사 기준
   - (d) 기타 (직접 입력)
```

## 모호 동사 케이스 (#1 NO)

사용자가 "정리해줘", "처리해줘", "봐줘" 같은 모호 동사로 요청한 경우, **#1을 구체화하는 질문을 맨 앞에 배치**한다:

```markdown
## 진행 전에 확인할 부분이 있어요

1. "정리"가 어떤 작업을 의미하시나요?
   - (a) 텍스트 추출 (원본 그대로 재구성)
   - (b) 표로 재구성 (항목별 분류)
   - (c) 현황 분석 (상태·인사이트 도출)
   - (d) 엑셀 변환 (구조화된 워크북)
   - (e) 기타 (직접 입력) ★

2. 어디에 집중하면 될까요?
   ...

3. 결과 파일 형태는?
   ...
```

## 이미 명확한 항목은 묻지 않음 (부분 YES 케이스)

예) "요약해줘"만 있는 경우: #1 YES, #2~#4 NO.
→ #1은 묻지 않고 #2~#4만 묶어서 질문:

```markdown
## 요약 범위·형태 확인드려요

1. 어디에 집중한 요약이 좋을까요?
   - (a) 전체 내용 핵심
   - (b) 채산·손익 부분만 ★
   - (c) 리스크·이슈 부분만
   - (d) 개선안·제안 부분만

2. 요약 분량·형태는?
   - (a) 3줄 핵심
   - (b) 1페이지 보고서 ★
   - (c) 표 형태 비교
   - (d) 슬라이드 노트

3. 이 요약은 누구를 위한 것인가요?
   - (a) 내부 팀 공유
   - (b) 임원 보고 ★
   - (c) 외부 고객
   - (d) 개인 참고
```

**주의:** "#1이 YES이므로 전체 질문을 스킵"은 v0.13에서 금지. 나머지 NO 항목은 반드시 묻는다.
