# 이해 확인 형식 예시 (Step 4)

**전 분기 필수.** 분기 1(바로 진행)·3(역할 위임)·4(에스컬레이션 핸드오프) 어느 경우에도 실행한다.

**질문 포맷:** 구조화된 질문 포맷을 사용한다 — 번호 매긴 선택지 + ★ 권장 옵션 표기.

## 기본 형식

```markdown
## 진행 전 확인

- 요청: ___
- 집중 범위: ___
- 결과물: ___
- 배경 정보: ___ (분석·전략·보고서·리뷰일 때 포함)
- 진행 방식: 1) ___ 2) ___ 3) ___
- 예상 소요: 약 ___

선택해주세요:
  - (a) 맞습니다, 이대로 진행해주세요 ★
  - (b) 수정이 필요합니다 (어떤 부분인지 알려주세요)
```

## 실제 예시 1 — 경쟁사 분석 보고서

```markdown
## 진행 전 확인

- 요청: 경쟁사 분석 보고서 작성
- 집중 범위: A사·B사 대비 우리 제품의 기능 비교
- 결과물: 표 형태 비교 + 1페이지 요약
- 배경 정보: 임원 보고용, Q2 제품 전략 수립 목적
- 진행 방식: 1) 기능 항목 정리 2) 3사 비교표 작성 3) 요약·시사점 도출
- 예상 소요: 약 10~15분

선택해주세요:
  - (a) 맞습니다, 이대로 진행해주세요 ★
  - (b) 수정이 필요합니다 (어떤 부분인지 알려주세요)
```

## 실제 예시 2 — 단순 추출 (분기 1, 체크리스트 전항목 YES)

전항목 YES라도 Step 4는 **필수**:

```markdown
## 진행 전 확인

- 요청: 계약서 3장 Open Items 추출 → 엑셀
- 집중 범위: 3장 "Open Items" 섹션만
- 결과물: 엑셀 파일 (열: 쟁점·현재상태·카테고리·합의필요사항)
- 진행 방식: 1) 3장에서 항목 식별 2) 열 구조화 3) 엑셀 저장·전송
- 예상 소요: 약 5분

선택해주세요:
  - (a) 맞습니다, 이대로 진행해주세요 ★
  - (b) 수정이 필요합니다 (예: 다른 열 추가, 다른 장도 포함 등)
```

## 실제 예시 3 — 에스컬레이션 핸드오프 전 확인 (분기 4)

Step 5 판정 결과가 에스컬레이션 핸드오프 조건에 해당할 때에도 Step 4로 사용자 확인 먼저:

```markdown
## 진행 전 확인 + 역할 설계 제안

- 요청: 규제 컴플라이언스 체계 구축 제안
- 집중 범위: 2025 개정 규제 대비 전사 절차 개편
- 결과물: 절차 설계서 + 역할·책임 매트릭스 + 리스크 매핑
- 배경 정보: 감독기관 제출용, 12월 1일 시행 예정

**판정 결과:** 6기준 간이 판정에서 high 4건·총점 16점 → 기존 역할로 커버 불가.
→ Product/Planning 핸드오프을 제안드려요.

선택해주세요:
  - (a) Product/Planning 핸드오프해서 전담 역할 설계 후 진행 ★
  - (b) 기존 팀원으로 일단 진행 (리스크 감수)
  - (c) 범위·결과물 재정의 (수정 필요)
```

## 수정 요청 처리

사용자가 (b) "수정 필요"를 선택하면:
1. 어느 부분을 수정해야 하는지 세부 질문 (해당 항목만)
2. 이 수정 질문은 **질문 라운드 2**로 카운트 (라운드 1을 이미 사용한 경우)
3. 수정 내용을 반영하여 Step 4 재실행

---

## ❌ Anti-Example — 분기 1에서 Step 4 스킵 (절대 금지)

아래는 **Fail 판정되는 잘못된 패턴**이다.

```markdown
[Brief Gatekeeper] 📋
- [x] Step 1: 문서 이해
- [x] Step 2: 체크리스트 판정 (4/4 YES)
- [x] Step 4: 이해 확인 (자명)     ← ❌ 체크박스만 찍고 스킵
- [x] Step 6: 실행 ✅

(바로 엑셀 파일 전달)        ← ❌ 사용자 응답 없이 실행
```

**위반 사항:**
- Step 4 체크박스를 "자명"으로 찍고 넘어감 → Fail
- 선택지 블록(a/b) 미제공 → Fail
- 사용자 응답 없이 결과물 전달 → Fail
- Step 4 턴과 실행 턴이 분리되지 않음 → Fail

**올바른 패턴 (위 '실제 예시 2' 참조):**
1. 체크리스트 판정 출력
2. Step 4 요약 박스 + 진행 방식 + 선택지 블록 제공
3. **사용자 응답 대기**
4. 응답 수신 후 실행
