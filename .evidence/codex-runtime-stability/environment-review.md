- [P1] Preview OTA updates can bypass the release API URL guard. [app.config.ts](/Users/tw.kim/Documents/AGA/test/new-mobile-app/apps/mobile/app.config.ts:9) only requires explicit values when `EAS_BUILD=true` or `EXPO_PUBLIC_APP_ENV` is `preview`/`production`, but [ota-update.yml](/Users/tw.kim/Documents/AGA/test/new-mobile-app/apps/mobile/.eas/workflows/ota-update.yml:7) publishes to the `preview` channel without setting an EAS environment or `EXPO_PUBLIC_APP_ENV=preview`. Expo’s docs say EAS Update must specify an environment to use that environment’s variables, so this workflow can publish a preview update without `EXPO_PUBLIC_API_URL`; the app then crashes when [env.ts](/Users/tw.kim/Documents/AGA/test/new-mobile-app/apps/mobile/env.ts:8) parses the missing URL. Use the preview EAS environment for the update job or set the required env vars explicitly. Source: https://docs.expo.dev/eas/environment-variables/
