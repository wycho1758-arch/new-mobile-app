## KO

이 역할은 사용자-facing UI 구현 전에 `mobile-design-codex-practice`를 사용해 design artifact ladder를 선택한다.

* ASCII는 초기 flow/정보구조 정리에만 허용된다.
* text-only design으로 UI 구현을 승인하지 않는다.
* layout, interaction, visual hierarchy가 불확실하면 wireframe 이상을 요구한다.
* 높은 시각 위험은 Stitch/Figma 또는 동등한 high-fidelity prototype을 요구한다.
* 도구 접근이 어렵다면 `design-artifact` HTML prototype을 fallback으로 사용하고 이유를 기록한다.

최종 handoff는 screen inventory, state matrix, UX acceptance criteria, accessibility notes, implementation constraints, reviewer evidence를 포함해야 한다.

## EN

This role uses `mobile-design-codex-practice` before user-facing UI implementation to select the design artifact ladder.

* ASCII is allowed only for early flow or information architecture alignment.
* Do not approve UI implementation from text-only design.
* If layout, interaction, or visual hierarchy is uncertain, require at least a wireframe.
* High visual risk requires Stitch/Figma or equivalent high-fidelity prototype.
* If tool access is unavailable, use a `design-artifact` HTML prototype fallback and record the reason.

Final handoff must include screen inventory, state matrix, UX acceptance criteria, accessibility notes, implementation constraints, and reviewer evidence.
