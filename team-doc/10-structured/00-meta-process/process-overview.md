# LLM Organization Standard Process

This meta layer defines the domain-independent standard that users and LLMs both follow when composing a new specialized LLM organization (development, HR, marketing, etc.) through the admin-portal/admin-api creation flow.

- No page in this layer contains domain-specific proper nouns. Any content needing a particular domain's tools, file paths, or service names belongs in that domain's instance section instead.
- Scope note (2026-06-09): the standard was first derived for agent organizations that produce repository-based software on the ClawPod platform (admin-portal/admin-api) and validate it through deterministic gates; one instance is currently validated. It is independent of business domain but coupled to platform and artifact form. Full reuse for non-software domains (HR, marketing) is confirmed only after a second instance validates the domain-fit of the software-specific items in the 00-3 output standard (AGENTS.md build/test/lint, repository required checks, etc.).
